# 隱形背包 RPG：之後分工流程

之後固定用這個方式合作：

1. 在 Codex 這裡處理修改
   - 調整網頁流程
   - 修改畫面排版
   - 修正圖片位置
   - 修正下載功能
   - 整合新圖片或新版本
   - 檢查本機檔案是否正常

2. Codex 完成後會告訴你
   - 哪些檔案有修改
   - 是否已經可以推上 GitHub
   - 如果需要測試，會提醒要先確認哪些地方

3. 到 Antigravity 推上 GitHub
   - 開啟資料夾：`C:\Users\alway\OneDrive\桌面\隱形背包RPG`
   - 檢查變更
   - Commit
   - Push 到 GitHub

4. GitHub Pages 自動更新
   - 推送完成後，GitHub Pages 會自動更新公開網站
   - 如果沒有立刻更新，通常等 1 到 3 分鐘再重新整理即可

## 分工原則

- Codex：負責改檔案、整理版本、修功能、確認本機結果。
- Antigravity：負責登入 GitHub、建立版本紀錄、推送到 GitHub、讓 Pages 上線。

## 重要提醒

- 正式入口與主程式檔案皆為 `index.html`。
- 已經取消原先跳轉的設計，請直接在 `index.html` 上進行後續的修改或版本更新。
- 因為使用 Git 進行版本控制，不需再透過更改檔名（如修正版6、7）來區分版本。
- 不要把 `.gh-cli`、登入代碼、暫存檔推上 GitHub。
